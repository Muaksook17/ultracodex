## 🚀 Getting started

When you first open ultracodex, you see a clean view of your workspace. The screen displays your active agent fleet. Before you start, enter your API key in the settings menu. This key allows the software to connect with the models that perform the heavy lifting.

Select a workflow from the library. Each workflow defines a set of rules for your agents. You can monitor the progress of these agents in the main window. If an agent hits a block, it notifies you immediately. You then decide how to move forward.

The builder agents create the initial plan, while verifier agents check the work for accuracy. This loop ensures the code meets your specific standards.

## 🛠 Features

*   **Agent Orchestration:** Manage multiple agents at once. The software coordinates their movements so they reach the finish line without errors.
*   **Workflow Engine:** Create custom roadmaps for your projects. You define the path, and the agents execute the plan step by step.
*   **Visual Board:** View your agent progress in real time inside the interface. Every action appears clearly on your screen.
*   **Token Management:** Keep an eye on your usage data. The software tracks how many tokens you spend on each task to prevent going over your budget.
*   **Parallel Execution:** Run separate tasks at the same time. This feature saves time by performing multiple checks in a single window.

## 💡 Best practices

Keep your tasks focused. Assign one primary objective to a fleet of agents for best results. This allows the verifiers to double-check the logic without distraction. If you use a large budget, monitor the early stages of the workflow carefully. Once the agents establish a steady pattern, you can let them run unattended. 

Save your workspace after every major session. This preserves your current settings and agent progress. If you close the app, your saved work stays ready for your next session.
